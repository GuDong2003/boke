一个刚开始的博客，最不需要的往往就是复杂后台。

你真正需要的是三件事：

- 一个顺手写作的地方。
- 一条稳定发布的路径。
- 一套半年后看起来依然清楚的结构。

这就是这个起步方案背后的想法。

## 这套配置优先考虑什么

- 用 Markdown 写文章，内容本身最清楚。
- 所有修改都进 Git，方便回看和备份。
- 先用静态站部署，省钱、省心、速度也够快。
- 后面要加搜索、评论、表单时，不需要推倒重来。

## 为什么用 Astro

Astro 对博客很合适，因为它默认输出静态页面。对个人站来说，这通常意味着更少的维护、更高的性能，以及更低的复杂度。

同时它也保留了组件、布局和内容集合这些能力，所以网站长大以后依然能保持清楚。

## 接下来做什么

把这些示例文章替换成你的内容，改好站点信息，接上 GitHub 和 Cloudflare Pages，就已经是一套真正能持续发布的博客流程了。
